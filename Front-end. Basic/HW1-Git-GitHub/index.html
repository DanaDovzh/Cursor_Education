Тема лекції: "VCS (Git, Github, gh-pages, FTP)"

Базові команди

  • git config --global user.name blabla - встановлює ім'я, яке буде відображатися у створювальних комітах;
  • git config --global user.email blabal@bla - встановлює адрес електроної пошти, яке буде відображатися у створювальних комітах;
  • git init - створює новий репозиторій в поточній папці;
  • git status - допомагає дізнатися поточний стан репозиторію;
  • git add - додає файли в поточний коміт;
  • git commit - допомагає зберегти всі дані у внутрішній базі, які були додані завдяки git add;
  • git log - показує історію коммітів, які були створенні в поточному репозиторію;
  • git checkout - використовуючи цю команду:
    • можна повернутися до задоного коміту;
    • переключатися між гілками;
  • git branch - завдяки цій команді можна наступні дії:
    • створити нову гілку;
    • побачити список гілок;
    • видалити гілку;
    • змінити ім'я гілки;
  • git push - завантажує всі зміни в локальному репозиторію у віддалений;
  • git pull - завантажує історію з віддаленого репозиторію;
  • git merge - змерджує(об'єднує) одну або декілька гілок в одну;
  • git clone - клонує(копіює) віддалений репозиторій.

А для чого потрібний .gitignore файл?

Такий файл слугує для того, щоб вказати в ньому які папки і файли потрібно приховати віддалений VSC Github. У файлі .gitignore в кожному рядку якого міститься шаблон файлу або каталогу, який необхідно проігнорувати.

Приклад рядків файлу:

# Object files
*.o
*.ko
*.obj
*.elf

Додаткові команди

  • git revert - створює ще один комміт "антикоміт" для конкретного коміту, таким чином скасовує зміни, які були виконані в ньому;
  • git reset - виконує скасування змін. Містить три основних флажка:
    • --soft@n - повертається назад на n коммітів, але при цьому зміни зберігаються;
    • --mixed@n- за замовчуванням;
    • --hard@n- повертається назад на n коммітів і видаляє зміни;
  • git rebase - дозволяє згрупувати всі коміти в один патч і перенестиміж гілками;
  • git cherry-pick - дозволяє перенести конкретний коміт з одної гілки в іншу, без використання git merge;
  • git stash- дозволяє зберегти зміни у поточному файлі в окреому сховищі, а сам файл повернути до початкого стану.
